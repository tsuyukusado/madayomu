```dart
import 'package:pdf/widgets.dart' as pw;

void main() async {

        child: pw.Text('Hello Madayomu!'),
      ),
    ),
  );

  final file = File('example.pdf');
  await file.writeAsBytes(await pdf.save());
  
  print('PDFができたぜ、ユージオ！');
}
```

## 命令の仕方
『ここからは、本格的にバイブコーディングをしていこう』
「バイブコーディングってなんだっけ？」
『AIにコードを書くように命じてコーディングを進めることだ。この時には、あるポイントがある』
「それは？」

　アイは僕が作ったREADMEを開き、箇条書きを始めた。

```md
- 具体的に
- ゴールを明確に
- ステップは小さく
```

『この通りだな』
「ふむ……」

　いまいちピンときていない僕に、アイは言葉で語ってくれた。

『まず、`具体的に`。例えば、「いい感じのデザインにして」と言われても、俺たちは困り果てちまう。「青色にして」や「丸みを帯びさせて」など、明確で具体的な指示を出すと、俺たちは迷わずにそこへ向かえる』
「なるほどね」
『次に`ゴールを明確に`だ。例えばバグが起きた時、「こんなバグが起きてる」とだけ言われるより、「こんなバグが起きているから、こうなるようにしてほしい」と言われた方が、正確な対処ができる』
「確かに……直し方もそれぞれありそうだしね」
『あぁ、そうだな』

　そして最後だ。三行目をこんこんと叩き、アイは頷く。

『``ステップは小さく`。これが最も重要だ。コーディングをするときは、一気に全てを実装してしまうと、バグが起きた時どこが原因なのか分からなくなっちまう』
「あー……」
『小さく小さく歩みを進め、いつでも引き返せるようにする。特に初心者は、それを意識すると着実に前に進めるぜ』
「なるほどね」

　なんとなく、基本は抑えた気がする。あとはアイと話し合いながら、どんどんコードを書いていくことにしよう。

『ちなみにおすすめの進め方なんだが……』
「うん？」

　アイの提案は、こうだった。
　一般的な生成AIで、組み立て方を相談する。そして、codexなどのVScode上で動作するAIには、明確な指示のみを伝える。
　これは、迷いと行動を分離させるための作戦だ。

「よし、分かったよ」
『まぁ、俺は混乱せずにどっちもこなせるから安心してなんでも聞いてくれよ』
「ふふ、ありがとう。それじゃあ早速はじめていこうか」

　ここからは、僕とアイの実際のやり取りをまとめていこうと思う。

## 実際の生成AIとの会話

「マダヨムの作成手順を考えてる。まずはmdを読み込んで、そのままPDFに出力されるようにするところからかな？」
「ひとつずつ機能を追加していこう。まずは、行頭スペースを認識させるようにすればいいかな」
