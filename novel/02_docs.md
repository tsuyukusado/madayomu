# 文字にして整理しよう

『これからお前には、２枚の書類を書いてもらう』
「書類……？」

　僕がVScodeの画面を見ながらぽかんと固まっていると、AIが凛とした態度でそう言った。

『今お前は、俺と一緒にマダヨムのコンセプトを固めたよな。でもそれは単なる｜会話《圏》だ。このままでは、後から読み返したり、コンセプトを見直したりすることができない」
「それはそうだね」
『だから書類にまとめる。今話し合ったことを、コンセプトとして文字に起こすんだ』
「え？　そんなこと出来るかな……」

　確かに僕の中にはつくりたいものがあるけれど、いざそれを言語化しようとすると、とても難しい気がする。果たしてそんなことが出来るだろうか……と再び意気消沈しかけた僕だったけれど、彼はそんな僕を見てくすくすと笑っていた。

「な……なんで笑っているの？」
『お前、今自分が誰と話しているのか、忘れてないよな？』
「え？」
『AIが最も得意としているのは文字だぜ？』
「……あ！」

　僕ははっとして彼の顔を凝視した。『任せとけ』と言ってにっと笑ったAIは、再びパソコンの中へと潜り込んでいく。新たにファイルが生成され、文字列が一気に入力されていく。

```md
# マダヨム　〜ワンクリックでPDFつくーる〜

## 悩みを抱える人のプロフィール
- 小説や文章を書くのが好き
- Web小説投稿サイト（例：カクヨム）で作品を書いている
- 自分の作品を「本」という形にしたい気持ちがある
- プログラミングに興味はあるが、専門家ではない
- ツールや技術の選定で手が止まりがち

## 抱えている悩み
- カクヨム記法で書いた原稿を、そのまま本にできない
- ルビや圏点をWordや既存の組版ツールで再現するのが大変
- マークダウンは便利だが、対応した組版ツールが少ない
- 「何から始めればいいのか分からない」ため、形にする前に挫折してしまう
- つくりたいものはあるのに、技術との間に大きな壁を感じている

## その解決策
- カクヨム記法とマークダウン、両方に対応した原稿入力を可能にする
- テキストをコピペするだけで、ルビ・圏点を正しく反映したPDFを生成する
- 難しい設定や専門知識を不要にし、「ワンクリック」で完結させる
- 個人出版・同人・保存用など、さまざまな用途で使えるPDFを出力する
- 「書くこと」に集中したまま、「本」という形まで辿り着ける体験を提供する

## コンセプトを一言でまとめると？
- ワンクリックで、出版できる本になるツール。
```

『どうだ？』

　AIがこちらへ振り返りはにかむ。
　僕は表示された文字を目で追いかけ、ほうっと感嘆の吐息を零した。

「すごい……まさにこの通りだよ……僕のコンセプトがぎゅっと詰まってる」
『これがAIの力だ』

　彼はぐっと胸を逸らして、自慢げな顔をする。僕は思わず、その様子にくすくすと笑ってしまった。

「ふふ、すごいや。……ねぇ、君のことはなんて呼んだらいい？」
『俺はAIだから、特別な名前は定まってないけど……』
「ふぅん……そうだな……」

　僕は瞳を瞬かせるAIの様子を見ながら、思考を巡らせた。
　安直だけど――



「それなら、君は今日からアイ！」



　――なーんて。
　照れくささに思わず笑ってしまった僕を、彼はぽかんと見ていたけれど。
　そのうち、「はははっ！」とお腹を抱えて笑った。

『安直すぎるだろ！　……でも気に入った。よし、俺のことはこれからはアイと呼べ』
「うん。よろしくね、アイ。それで……君はさっき、書類を**２つ**作るって言ってなかった？」
『よく覚えていたな。そのとおりだ、あと１つ、書類が残っている』

　AIことアイは、真剣な顔をして改めてパソコンを見た。

『お前が今回のマダヨムというツールを作るにあたって、一番最初に実現したいものっていうのはなんだ？』
「……え？」

　僕がぽかんとしていると、アイは『そうだな……』と腕を組む。

『お前は今回のツールに、色々な機能を入れたいと思っているだろう？』
「それは、そうだね」
『でも、それらを全部、１から盛り込んでいたら、ツールはなかなか完成しない。完成しなければ、それが本当にお前の理想とするツールなのか、実感することも出来ずに挫折してしまうかもしれない』
「それは確かに……」
『だから今回の２つ目の書類では、まず最初に何をどこまで作るのか、それを決めるんだ。まだこの機能は要らない、と思う部分は徹底的に削ぎ落として、必要最低限の形で完成までこぎつける。それが大切なんだよ』
「そうなんだ」

　うーん、と、僕は頭を悩ませる。実現したい機能はたくさんあるけれど、どれが一番大事かと言われると難しい。

『例えば、だ』

　悩む僕の顔を見て、アイが声を上げた。

『そもそも最初は、使うのはお前だけだよな？』
「うん」
『となると、**ネットに公開する必要はない**よな？』
「……確かに」

　こく、と頷いた僕を見て、アイは『その調子で考えればいい』と笑う。
　再びパソコンの中へ潜っていった彼と話し合いながら、僕が作り出した書類は以下の通りだ。

```md
#　今回作るもの
- カクヨム記法に対応する
    - ルビ
    - 圏点
- マークダウン記法に対応する
    - 見出し
    - 箇条書き
    - 太字
    - インラインコード
    - コードブロック
- PDFを生成する
    - 表紙を生成する
    - ページ番号を振る
    - 目次を生成する
    - 奥付を生成する
    
# 今回作らないもの
- その他のマークダウン記法
    - 理由：僕が使わないから
- 裏表紙、見開き表紙の生成
    - 理由：まず電子書籍から始めるから
- 実際に操作する画面の実装
    - 理由：まずは僕が使えればいいから
```

『いい調子だな』

　アイがにっと笑う。

『これでいいんじゃないか？　今回何を作ればいいのかが、かなり明確になっただろう』
「うん、そうだね。これなら始められるような気がする」

　たった２つのシンプルな書類を作っただけなのに、次に僕がやるべきことが整理された――そんな気がした。

---

　つくらないものを決めるときは、**その理由**も明記しておくのが重要なんだって。
　後から振り返って読んだとき、「なんでこれを抜いたんだっけ？」って忘れがちだから。
